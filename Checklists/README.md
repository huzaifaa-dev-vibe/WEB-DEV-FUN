# Checklists

> Every checklist you need. Run before shipping anything.

---

## Purpose

Codified checklists for every stage: project setup, pre-flight, pre-launch, post-mortem, accessibility, performance, security.

## Prerequisites

- A project you're shipping.

## Learning Outcome

You can confidently say "we didn't forget anything major".

## AI Instructions

When shipping:
1. Run [pre-flight.md](pre-flight.md) before opening a PR.
2. Run [pre-launch.md](pre-launch.md) before going live.
3. Run [accessibility.md](accessibility.md) on any UI change.
4. Run [performance.md](performance.md) on any UI change.
5. Run [security.md](security.md) on any auth/encryption change.
6. Run [post-mortem.md](post-mortem.md) after every incident.

## Anti-patterns

- "I'll run the checklist later" (you won't).
- Skipping checklist because "this is a small change".
- Treating checklist as bureaucratic — they're for you, not for compliance.
