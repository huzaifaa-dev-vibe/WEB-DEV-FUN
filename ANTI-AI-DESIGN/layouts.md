# Anti-AI-Design: Layouts

> Layout patterns beyond the 3-card grid.

---

## The AI tells

1. 3-card "Features" grid (icon + headline + 1 sentence) × 3.
2. Centered hero + 2 CTAs.
3. Testimonial cards × 3.
4. Pricing 3-tier table.
5. FAQ accordion.
6. Footer with 4 columns of links.

All in the same order, on every SaaS site.

---

## Human approach

### Break the grid
- Some sections full-bleed.
- Some sections break to 1 column.
- Some elements overlap.

### Vary section types
- Not every section is a card grid.
- Use: big text, single image, table, video, quote, code block, list, timeline.

### Tell a story
- Hero: promise.
- Section 2: proof (one specific case).
- Section 3: how it works (real diagram, not icons).
- Section 4: more proof (real numbers).
- Section 5: pricing (or skip — make them contact).
- CTA: simple.

Not: Hero → Features → Testimonials → Pricing → FAQ.

---

## Layout ideas

### Big text section
- Just a sentence, big.
- No card, no icon, no image.
- Powerful when used sparingly.

### Single product screenshot
- Full-width screenshot.
- No text wrapping around.
- Just the product, big.

### Side-by-side comparison
- Old way vs new way.
- Real screenshots, not icons.

### Timeline
- Horizontal or vertical.
- Milestones with real dates.

### Code block
- Real code that does something.
- Syntax highlighted.
- Sometimes more compelling than any feature card.

### One big quote
- Customer quote, full bleed.
- Massive text.
- Link to case study.

### Stat grid
- 3-4 big numbers.
- No icons, no cards.
- Just numbers + tiny labels.

### Table
- Real comparison table.
- More honest than marketing fluff.

### Video
- Real product video.
- Muted autoplay, controls on hover.

### FAQ inline
- Not accordion. Just questions as headings, answers below.
- More scannable.

---

## Section ordering (alternative)

Instead of:
```
Hero → Features → Testimonials → Pricing → FAQ → CTA
```

Try:
```
Hero → Big promise → How it works (real diagram) → One case study → Pricing → CTA
```

Or:
```
Hero → One stat → Real screenshot → Customer logos → Pricing → CTA
```

Or (manifesto style):
```
Big statement → Why we built this → How it's different → Try it
```

---

## Examples to study

- Linear: long-form hero, minimal sections, big screenshots.
- Stripe: dense, varied sections, real product everywhere.
- Vercel: minimal, code-focused.
- Apple: product photo dominant, minimal text.
- Notion: playful, illustration + screenshot mix.
- Figma: colorful, real product UI.

None of them are "centered hero + 3 cards + testimonials + pricing".
